# Eternal Foundation

This repository is hackathon-facing, but it is built on work completed and tested during Colosseum Eternal.

## What Was Built During Eternal

- attribution logic for funding provenance, control exposure, permissions, liquidity behavior, and recurrence
- decision outputs shaped around `ALLOW / REVIEW / BLOCK`
- operator-facing evidence packaging
- MCP-facing machine decision delivery
- AEP framing for constitutional execution controls
- case review, report flow, and liability-oriented workflow concepts
- repeated runtime hardening and live-asset testing

## Why That Matters

The Eternal sprint reduced the amount of “from-scratch” product risk in this hackathon.

Instead of starting from a blank repo, the hackathon effort can focus on:

- product clarity
- stronger AEP framing
- cleaner public infrastructure narrative
- judge-facing presentation
- sharper boundaries between attribution, MCP, and constitutional execution
